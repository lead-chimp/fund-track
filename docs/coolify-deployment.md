# Coolify Deployment Guide for Fund Track

This document provides a comprehensive guide for deploying the Fund Track application using Coolify.

## Prerequisites

- Coolify instance installed and running
- GitHub repository access
- Domain name ready: `fund-track.merchantfunding.com`
- Environment variables prepared

## Overview

Coolify will handle:
- Automated deployments from Git
- SSL certificate management (Let's Encrypt)
- Database management (PostgreSQL)
- Process management and monitoring
- Load balancing and scaling
- Backup management

## Step 1: Coolify Server Setup

### 1.1 Install Coolify (if not already done)

```bash
# On your Coolify server
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash
```

### 1.2 Access Coolify Dashboard

Navigate to your Coolify instance:
- URL: `https://your-coolify-server.com`
- Complete initial setup and create admin account

## Step 2: Project Setup in Coolify

### 2.1 Create New Project

1. Login to Coolify dashboard
2. Click "New Project"
3. Name: `Fund Track`
4. Description: `Lead management and intake workflow application`

### 2.2 Add Git Repository

1. In your project, click "New Resource"
2. Select "Application"
3. Connect your GitHub App
4. Select the Fund Track repository
5. Branch: `main`
6. Application name: `fund-track-app`

## Step 3: Application Configuration

### 3.1 Build Settings

Accept the default build settings provided by Coolify. The application will use the standard Next.js build process.

### 3.2 Environment Variables

Add all required environment variables in Coolify's environment configuration section. This includes database credentials, API keys, and application-specific settings.

## Step 4: Database Setup

### 4.1 Add PostgreSQL Service

1. In your project, click "New Resource"
2. Select "Database"
3. Choose "PostgreSQL"
4. Configuration:
   - Name: `fund-track-postgres`
   - Version: `17` (Coolify default)
   - Database name: `fund_track_prod`
   - Username: `fund_track_user`
   - Password: Generate secure password

### 4.2 Database URL

The `DATABASE_URL` environment variable will be automatically generated by Coolify when the PostgreSQL service is created and linked to your application.

### 4.3 Database Operations

Database operations (migrate, generate, seed) are performed manually after the build process is complete. These are not part of the automated build pipeline.

#### Manual Database Operations Workflow

1. After deployment, run migrations manually:
```bash
# SSH into your container or use Coolify's terminal
npm run db:migrate:prod
```

2. Generate Prisma client when needed:
```bash
npm run db:generate
```

3. Seed database (ONLY for initial setup):
```bash
# Safe seeding - will fail if database has data or in production
npm run db:seed

# Production seeding - creates only essential admin user
ADMIN_EMAIL=admin@merchantfunding.com ADMIN_PASSWORD=secure_password FORCE_SEED=true npm run db:seed:prod

# Force full seeding (DANGEROUS - deletes all data)
FORCE_SEED=true npm run db:seed:force
```

**Safety Features**: The seed scripts include built-in protections:
- Development seed automatically prevents running in production unless explicitly forced
- Production seed only creates essential admin user and checks if one already exists
- Checks for existing data and refuses to run if database is not empty
- Requires `FORCE_SEED=true` environment variable to override safety checks
- Adds delays in production to allow cancellation

## Step 5: Domain and SSL Configuration

### 5.1 Configure Domain

1. In your application settings, go to "Domains"
2. Add domain: `fund-track.merchantfunding.com`
3. Enable "Generate SSL Certificate" (Let's Encrypt)
4. Enable "Force HTTPS Redirect"

### 5.2 DNS Configuration

Update your DNS records:

```bash
# A Record
fund-track.merchantfunding.com    A    [COOLIFY_SERVER_IP]

# Optional: CNAME for www
www.fund-track.merchantfunding.com    CNAME    fund-track.merchantfunding.com
```

## Step 6: Deployment Configuration

### 6.1 Build Configuration

The application uses the standard Next.js build process. Prisma operations (migrate, generate, seed) are handled separately from the build process and performed manually after deployment.



## Step 7: Advanced Configuration

### 7.1 Resource Limits

Configure resource limits in Coolify:

```yaml

Memory Limit: 1Gi

```

### 7.2 Scaling Configuration


## Step 8: Monitoring and Logging

### 8.1 Application Logs

Coolify provides built-in log aggregation:
- Access logs through Coolify dashboard
- Real-time log streaming
- Log retention and search

### 8.2 Monitoring Setup

Configure monitoring:

```yaml
# Monitoring Configuration
Enable Metrics: true
Enable Alerts: true
Alert Channels:
  - Email: admin@merchantfunding.com
  - Slack: #fund-track-alerts (if configured)

Alert Rules:
  - High CPU Usage (>80% for 5 minutes)
  - High Memory Usage (>90% for 5 minutes)
  - Application Down (health check fails)
  - High Error Rate (>5% for 10 minutes)
```

## Step 9: Backup Configuration

### 9.1 Database Backups

Configure automatic database backups:

```yaml
# Backup Configuration
Database Backup:
  Enabled: true
  Schedule: "0 2 * * *"  # Daily at 2 AM
  Retention: 30 days
  Storage: S3 or local storage
```

### 9.2 Application Backups

```yaml
# Application Backup
File Backup:
  Enabled: true
  Include: logs, uploads (if any)
  Schedule: "0 3 * * 0"  # Weekly on Sunday at 3 AM
  Retention: 4 weeks
```

## Step 10: Deployment Process

### 10.1 Initial Deployment

1. **Verify Configuration**: Double-check all environment variables
2. **Deploy**: Click "Deploy" in Coolify dashboard
3. **Monitor**: Watch deployment logs for any issues
4. **Test**: Verify application is accessible at your domain

### 10.2 Deployment Pipeline

Coolify can be configured for automatic deployments:

```yaml
# Auto-deployment Configuration
Trigger: Git Push to main branch
Build Timeout: 15 minutes
Deploy Timeout: 5 minutes
Zero Downtime: true
Rollback on Failure: true
```

## Troubleshooting

### Common Issues

1. **Build Failures**
   - Check Node.js version compatibility
   - Verify all dependencies are listed in package.json
   - Review build logs in Coolify

2. **Database Connection Issues**
   - Verify DATABASE_URL format
   - Check PostgreSQL service status
   - Review database logs

3. **SSL Certificate Issues**
   - Verify DNS records are correct
   - Check domain configuration in Coolify
   - Review Let's Encrypt logs

4. **Performance Issues**
   - Monitor resource usage
   - Adjust CPU/memory limits
   - Consider scaling up replicas
